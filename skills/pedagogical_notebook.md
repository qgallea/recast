# /pedagogical-notebook — Generate a Shareable Jupyter Notebook

Generate a polished, self-contained Jupyter notebook from a completed RECAST
project. The notebook is designed for sharing online, teaching, and transparency.

## Usage
```
/pedagogical-notebook <project_path>
```

## What you do

### Step 1: Validate

Check that the project has completed the full pipeline:
- `<project_path>/.dml_project` exists
- `<project_path>/paper/review_history/final_report.md` exists
- `<project_path>/code_run/` contains executed notebooks
- `<project_path>/data/results/replication_check.json` exists

If any are missing, tell the user to run `/recast` first. Stop.

### Step 2: Read project context

Read these files to understand what the notebook should contain:
- `config.yaml` — slug, paper metadata, DML settings
- `data/paper_spec.json` — identification strategy, variables, main results
- `data/results/replication_results.json` — OLS replication output
- `data/results/replication_check.json` — pass/fail summary
- `data/results/dml_results.json` (if exists) — all DML estimates
- `data/results/hte_results.json` (if exists) — BLP, GATE, CLAN
- `data/results/causal_forest_results.json` (if exists) — CF estimates
- `data/results/diagnostics_flags.json` — diagnostic checks
- `paper/review_history/final_report.md` — final verdict and summary
- `code_run/03_replication.ipynb` — working replication code to extract
- `code_run/04_dml_extension.ipynb` or `code_run/04_causal_forest.ipynb` — working DML/CF code
- `data/dataset.parquet` — verify column names and shape

### Step 3: Create notebook directory

```bash
mkdir -p <project_path>/notebook
```

### Step 4: Generate the notebook

Write a Jupyter notebook (`.ipynb` JSON format) to `<project_path>/notebook/recast_<slug>.ipynb`.

The notebook has **8 sections**. Each section is a markdown cell followed by one or more code cells.

## Code extraction rules

When extracting code from `code_run/` notebooks:
- Remove `from paths import *` — replace with `PROJECT_DIR` paths
- Remove `matplotlib.use('Agg')` — the pedagogical notebook shows plots inline
- Remove LaTeX table generation (`\begin{tabular}`) — use `DataFrame.style` instead
- Remove JSON writing code — the notebook reads from existing JSON, doesn't rewrite it
- Keep regression fitting and DML fitting code intact
- Keep data cleaning and subsetting logic intact
- Simplify variable names where possible (e.g., `obj_rf` → `dml_forest`)

## Markdown writing rules

- Write for a **graduate student** who knows basic econometrics but not DML
- Explain **why before how** in each section
- Define terms on first use: cross-fitting, nuisance function, Neyman orthogonality, GATE, CLAN, BLP
- Reference the original paper by citation
- Include **interpretation after every result** — don't just show numbers, explain what they mean
- Keep explanations to 2-4 sentences per concept — not a textbook chapter

## Visualization rules

- `matplotlib` + `seaborn` for all plots
- Consistent style: `plt.style.use('seaborn-v0_8-whitegrid')`
- Every figure has: title, axis labels, legend (if multiple series)
- Forest plots: horizontal, with error bars, color-coded (grey=OLS, blue=DML, green=CF)
- Tables: `DataFrame.style` with `background_gradient` or conditional coloring
- Save all figures to `notebook/figures/` as well as showing inline

## Self-containment rules

- First code cell defines `PROJECT_DIR` as a single path the user can change
- All paths derived from `PROJECT_DIR`
- All imports in Section 2 cell 1
- No shell commands, no `!pip install`
- Must run end-to-end with "Run All" in Jupyter

## Step 5: Execute the notebook

After writing the notebook, execute it:

```bash
mkdir -p <project_path>/notebook/figures
cd <project_path>/notebook && \
jupyter nbconvert --to notebook --execute \
  --ExecutePreprocessor.timeout=3600 \
  --ExecutePreprocessor.kernel_name=python3 \
  --output recast_<slug>_executed.ipynb \
  recast_<slug>.ipynb
```

If execution fails, report the error and stop. The user needs to fix the issue.

## Step 6: Check verification

Read `notebook/verification_report.json` (written by Section 8 during execution).

If `status == "PASS"`:
