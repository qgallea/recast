## Referee 3 Report: Robustness and Replication Integrity
**Round:** 2
**Overall verdict:** Major concerns

---

### Blocking issues

- **Preferred DML coefficient reported in the paper does not match `dml_results.json`.** The abstract, Section 3.2, and the conclusion all state the preferred LassoCV estimate as $\hat\theta = -23.19$ (95 % CI: $[-34.27,\,-12.11]$, $p < 0.001$). However, `dml_results.json` records `preferred_coef: -25.470501740757953`, `preferred_ci_lo: -36.53090237829629`, `preferred_ci_hi: -14.410101103219619`. The discrepancy is approximately 10 % on the point estimate (−23.19 vs. −25.47) and the CI bounds differ by roughly 2 units in each tail. The paper's headline result is not traceable to the current state of the JSON output file, which violates the basic traceability requirement. Either the paper was compiled from an earlier JSON snapshot (pre-revision) and was not recompiled after the dml_results.json update, or the numbers were manually edited. This must be corrected: the paper text, abstract, and tables must match the current `dml_results.json` exactly.

---

### Major issues

- **Forest plot caption still misrepresents figure content.** The figure caption in Section 3.2 states: "OLS (Tables 1, 3, 6, 7), 2SLS (Table~2), and DML-PLIV (LassoCV)." The regenerated forest plot, however, shows exactly two OLS rows (Table 6, $n = 143$; Table 7, $n = 106$) and one DML-LassoCV row. Tables 1, 3 (historical OLS), and Table 2 (2SLS) are absent from the figure. This was flagged in Round 1 as a discrepancy between caption and image; it remains unresolved. The caption must be corrected to list only the rows actually shown: "OLS (Tables 6, 7) and DML-PLIV (LassoCV)." Alternatively, if the intent is to show all six targeted specifications, the historical OLS and 2SLS rows must be added to the plot. As it stands, a reader comparing the caption to the figure will encounter a direct contradiction.

- **Diagnostics table in paper reports stale and internally inconsistent values.** The diagnostics table (Section 4) lists `Nuisance quality: FAIL` with `R²_D = -127,810`. This is the value explicitly flagged in `dml_results.json` as `original_rf_r2_treatment_buggy` — a computation artifact from a pre-revision misaligned averaging step. The current JSON records `nuisance_r2_treatment: null` for RF (degenerate, excluded) and `nuisance_r2_treatment: -97090.74` for LassoCV (also a DoubleML artifact, but a different value). Reporting the superseded buggy figure in the published table, while the JSON notes document it as erroneous, is an internal inconsistency. The table should be updated to reflect the current diagnostics: RF treatment nuisance degenerate (excluded); LassoCV treatment R² is a DoubleML reporting artifact unrelated to model quality. Furthermore, the diagnostics table labels the DML direction check as `FAIL` with value $-23.19$, perpetuating the stale coefficient. The `diagnostics_flags.json` `dml_direction` check correctly uses the value $-25.47$ from the current JSON; the paper table does not.

- **Forest plot label for Table 7 shows $n = 106$, inconsistent with paper text and `paper_spec.json`.** The forest plot labels the Table 7 OLS row "OLS Full Controls (Table 7, n=106)." The paper text (Section 2.2, baseline contemporary specification) states "$N = 106$" in one place but the abstract says "$N = 106$" and the `paper_spec.json` records `"n_obs": 109` for Table 7 col 5. The discrepancy between $n = 106$ and $n = 109$ is unexplained. If the replication uses a slightly different filter, this should be documented; if it is a label error on the plot, it must be corrected. Mismatched sample sizes across outputs undermine numerical traceability.

---

### Minor issues

- **N = 148 for DML versus N = 143 for the nearest OLS baseline remains unexplained.** `dml_results.json` records $N = 148$. The closest OLS baseline (Table 6) uses $N = 143$. The paper does not explain the 5-observation gap. This was flagged in Round 1 and was not addressed. A single sentence explaining that the DML run drops the quadratic term (`pdiv_aa_sqr`) and thereby retains observations missing from the OLS quadratic specification would suffice.

- **`replication_check.json` applies a flat 5 % threshold inconsistent with the two-tier rule stated in the paper.** The JSON summary reads "4/6 specs replicated within 5%." The paper states a 5 % threshold for pure OLS but 10 % for bootstrap-SE or IV specifications. Two of the six targeted specifications use bootstrap SEs (Tables 3 col 5, Table 3 col 6). It is unclear whether those two pass under the 10 % rule; if they do, the pass count should be higher than 4/6, or the 5 % flat threshold in the pipeline check needs to be reconciled with the stated two-tier rule. This should be clarified in the diagnostics table note.

- **No within-paper robustness check for the preferred DML estimate.** The conclusion acknowledges "richer instrument sets or ensemble learners" as future work, but no sensitivity analysis is presented for $\hat\theta = -25.47$ (current JSON value): not to the number of folds, not to the control set, not to dropping continent fixed effects. With only one functioning learner (LassoCV) and a sample of $N = 148$, a minimal check — e.g., $K = 3$ folds or one alternative control specification — would materially strengthen confidence in the preferred estimate.

### Comments to the authors

**On the critical number mismatch.** The most urgent issue in this round is that the headline DML result printed in the abstract ($\hat\theta = -23.19$, CI $[-34.27, -12.11]$) does not match `dml_results.json` ($\hat\theta = -25.47$, CI $[-36.53, -14.41]$). The discrepancy is not a rounding artefact — it is a 10 % gap on the point estimate. Given that the revision updated `dml_results.json` (adding the `rf_nuisance_degenerate` flag and correcting the rep-averaging bug), it appears the paper was not recompiled from the updated JSON. All references to the preferred DML coefficient and CI must be updated to match the current data file before the paper can be accepted. The traceability chain — JSON → LaTeX table → paper text — must be unbroken.

**On forest plot consistency.** The regenerated forest plot is a clear improvement: the RF row has been removed and the figure is now readable, with the LassoCV CI legible at the left of the x-axis. This resolves the core presentational defect from Round 1. However, the caption continues to promise rows for Tables 1, 3, and 2SLS that are not present in the image. A reader consulting the caption will be misled. The fix is simple: update the caption to "OLS (Tables 6, 7) and DML-PLIV (LassoCV)" and add a parenthetical noting that historical specifications are reported in Table 1 of the paper but omitted from the forest plot because they use a different outcome variable. Note also the $n = 106$ vs. $n = 109$ discrepancy on the Table 7 label, which should be verified and corrected to match `paper_spec.json`.

**On the diagnostics table.** The diagnostics table in Section 4 is currently internally inconsistent in three ways: it reports the superseded buggy $R^2_D = -127{,}810$ rather than the current value, it uses the stale coefficient $-23.19$ for the DML direction check, and it labels the DML direction check as `FAIL` even though the paper explicitly argues (correctly) that the sign change is expected and not a model failure. The authors should consider relabelling this check as `WARN (expected)` in the table, consistent with the language already used in `diagnostics_flags.json`, and update all numeric values to match the current JSON outputs.

**On replication fidelity.** The Round 1 concern about the 21-country failures has been addressed qualitatively: the paper now explains the Conley SE unavailability and small-$n$ fragility, which is the correct substantive response. These failures are confined to the secondary historical specifications and do not undermine the main contemporary findings. The four passing specifications — including the two large-sample historical OLS regressions and the two contemporary OLS regressions — adequately cover the paper's primary claims. No further quantitative sensitivity analysis is required for acceptance, though it would strengthen the paper. The replication pass/fail threshold logic in the pipeline code (flat 5 % vs. stated two-tier rule) should be clarified in the diagnostics note for transparency.
